---
title: 'Get Latest Medical Record'
api: 'GET /api/medical-info/latest'
description: 'Retrieve the most recent medical record for a user'
---

## Authentication

This endpoint requires authentication via JWT Bearer token. Users can only retrieve their own medical records.

## Query Parameters

<ParamField query="userId" type="number" required>
  The ID of the user whose medical record you want to retrieve. Must match the authenticated user's ID.
</ParamField>

## Response

<ResponseField name="id" type="number">
  The unique ID of the medical record.
</ResponseField>

<ResponseField name="user_id" type="number">
  The ID of the user this record belongs to.
</ResponseField>

<ResponseField name="glucose" type="number">
  Blood glucose level (mg/dL).
</ResponseField>

<ResponseField name="oxygen_blood" type="number">
  Blood oxygen saturation level (%).
</ResponseField>

<ResponseField name="blood_pressure_systolic" type="number">
  Systolic blood pressure (mmHg).
</ResponseField>

<ResponseField name="blood_pressure_diastolic" type="number">
  Diastolic blood pressure (mmHg).
</ResponseField>

<ResponseField name="temperature" type="number">
  Body temperature (°C or °F).
</ResponseField>

<ResponseField name="age" type="number">
  Age of the user in years.
</ResponseField>

<ResponseField name="height" type="number">
  Height (cm or inches).
</ResponseField>

<ResponseField name="weight" type="number">
  Weight (kg or lbs).
</ResponseField>

<ResponseField name="respiratory_rate" type="number">
  Respiratory rate (breaths per minute).
</ResponseField>

<ResponseField name="blood_type" type="string">
  Blood type (e.g., A+, O-, AB+, etc.).
</ResponseField>

<ResponseField name="heart_rate" type="number">
  Heart rate (beats per minute).
</ResponseField>

<ResponseField name="created_at" type="string">
  Timestamp when the record was created (ISO 8601 format).
</ResponseField>

## Authorization

The authenticated user can only retrieve medical records for their own `userId`. Attempting to access records for a different user will result in a 403 Forbidden error.

## Notes

- This endpoint returns only the **most recent** medical record based on the `created_at` timestamp.
- If no medical records exist for the user, a 404 error is returned.
- All field names in the response use snake_case (database column naming convention).
